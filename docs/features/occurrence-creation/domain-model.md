## Aggregate: Occurrence (root)

Already scaffolded — extend for creation factory.

```text
Occurrence (aggregate root)
├── id: OccurrenceId (uuid v7 — generated by DB or domain)
├── cityId: CityId
├── category: OccurrenceCategory (VO)
├── occurrenceKind: OccurrenceKind (VO)
├── status: OccurrenceStatus (VO) — starts unverified
├── confidenceLevel: ConfidenceLevel (VO) — starts 0
├── problemLocation: ProblemLocation (VO)
├── storedMapLocation: MapLocation (VO) — may differ after privacy shift
├── privacyLevel: PrivacyLevel (VO)
├── description: OccurrenceDescription | null
├── contributorRef: ContributorRef (from Identity)
├── authorDisplayPolicy: AuthorDisplayPolicy (from Identity policy)
├── isSensitive: boolean (derived from category)
├── version: number
├── createdAt, updatedAt
└── deletedAt: null on create
```

### Factory method (creation only)

```typescript
// Conceptual API
Occurrence.createNew(params: {
  cityId: CityId;
  category: OccurrenceCategory;
  occurrenceKind: OccurrenceKind;
  problemLocation: ProblemLocation;
  privacyLevel: PrivacyLevel;
  description?: OccurrenceDescription;
  contributorRef: ContributorRef;
  authorDisplayPolicy: AuthorDisplayPolicy;
  idGenerator: () => OccurrenceId;
  clock: () => Date;
  locationPrivacy: LocationPrivacyPort;
}): { occurrence: Occurrence; event: OccurrenceCreatedEvent }
```

`createNew` enforces INV-O1 through INV-O10. Use existing `create()` only for rehydration from persistence.

| `OccurrenceDescription` | max 2000, no doxxing | Optional |
| `ContributorRef` | from Identity module | `reputationId` |
| `AuthorDisplayPolicy` | from Identity module | includes `forced_ghost` |

## Domain services

### `LocationPrivacyService` (or port)

```typescript
applyForStorage(
  problemLocation: ProblemLocation,
  privacyLevel: PrivacyLevel,
): MapLocation
```

| `privacyLevel` | Behavior |
|----------------|----------|
| `public` | Same as problem |
| `neighborhood` | Same stored coords; read model masks (not creation concern) |
| `approximate` | Random offset 100m default |
| `hidden` | Store coords; map layer excludes pin |

Implementation may live in `packages/geo` with domain port wrapper.

### `OccurrenceCategoryRegistry` (port)

```typescript
isAllowed(cityId: CityId, category: OccurrenceCategory): boolean
isSensitive(category: OccurrenceCategory): boolean
defaultKind(category: OccurrenceCategory): OccurrenceKind
```

Adapter: static config JSON or DB table `city_categories`.


### Handler responsibilities

1. Resolve session → `ContributorRef`
2. Validate Zod input (strict)
3. Resolve `cityId` (Rule C2)
4. Call `Occurrence.createNew`
5. `repository.save`
6. Publish event / outbox
7. Map to `OccurrenceCreatedDto` (strip sensitive author)

## Shared schema changes (`packages/shared`)

Current schema accepts `cityId` in body — keep but document validation in handler.

```typescript
// Planned additions
export const createOccurrenceSchema = z
  .object({
    cityId: z.string().uuid().optional(),
    category: z.string().min(1).max(64),
    description: z.string().max(2000).optional(),
    latitude: z.number().min(-90).max(90),
    longitude: z.number().min(-180).max(180),
    privacyLevel: z
      .enum(['public', 'neighborhood', 'approximate', 'hidden'])
      .default('public'),
    occurrenceKind: z.enum(['problem', 'temporary_event']).optional(),
  })
```

## Database changes (migration 0002 — planned)

Extend `0001_initial_occurrences.sql` concept:

```sql
ALTER TABLE occurrences
  ADD COLUMN contributor_reputation_id UUID NOT NULL,
  ADD COLUMN occurrence_kind TEXT NOT NULL DEFAULT 'problem',
  ADD COLUMN is_sensitive BOOLEAN NOT NULL DEFAULT false,
  ADD COLUMN author_display_policy TEXT NOT NULL DEFAULT 'ghost',
  ADD COLUMN description TEXT,
  ADD COLUMN map_latitude DOUBLE PRECISION,
  ADD COLUMN map_longitude DOUBLE PRECISION;
  -- map_* may equal lat/lng or shifted; hidden still stored for stats
```

**Still no** `contributor_latitude` / `contributor_longitude`.

Indexes:

- `(city_id, created_at DESC)` — admin lists
- GiST on `ST_MakePoint(map_longitude, map_latitude)` — Phase 3 read path

RLS: insert only when `city_id = current_setting('app.city_id')`.
