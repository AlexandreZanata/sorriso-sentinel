# Mobile i18n

Internationalization for end-user copy in the React Native app. **Code and keys remain English** per project language policy.

## Locales

| Locale code | Language | Status |
|-------------|----------|--------|
| `en` | English | Default |
| `pt-BR` | Brazilian Portuguese | Planned |

Future locales add a JSON file under `src/i18n/locales/` — no code changes outside `locale.ts` allowlist.

## Key naming

Use dot-separated English paths — never Portuguese in keys:

```text
occurrence.create.title
occurrence.create.submit
auth.login.emailLabel
errors.networkUnavailable
errors.rateLimitExceeded
```

Typed constants in `src/i18n/keys.ts`:

```typescript
export const I18N_KEYS = {
  occurrence: {
    create: {
      title: 'occurrence.create.title',
      submit: 'occurrence.create.submit',
    },
  },
} as const;
```

## File layout

```text
apps/mobile/src/i18n/
├── locales/
│   ├── en.json
│   └── pt-BR.json
├── keys.ts
├── translate.ts
├── locale.ts
└── use-translation.ts      # React hook wrapping translate + locale
```

### Example locale file (`en.json`)

```json
{
  "occurrence.create.title": "Report a problem",
  "occurrence.create.submit": "Submit report",
  "auth.login.emailLabel": "Email",
  "errors.networkUnavailable": "Could not reach the server. Check your connection."
}
```

### Example locale file (`pt-BR.json`)

```json
{
  "occurrence.create.title": "Reportar um problema",
  "occurrence.create.submit": "Enviar report",
  "auth.login.emailLabel": "E-mail",
  "errors.networkUnavailable": "Não foi possível conectar ao servidor. Verifique sua conexão."
}
```

## Runtime resolution

1. Read persisted preference from AsyncStorage (`sentinel-mobile-locale`).
2. Fall back to device locale via `expo-localization` (`pt*` → `pt-BR`, else `en`).
3. Expose `setLocale()` in Settings; re-render via React context.

Mirror the API docs pattern in `apps/api/src/features/docs/i18n/` — separate message catalogs per locale, single `t(locale, key, params)` function.

## Interpolation

Support `{name}` placeholders — same param names in every locale file:

```json
{
  "validation.votesRemaining": "{count} votes remaining this hour"
}
```

```typescript
t('validation.votesRemaining', { count: 12 });
```

## What is not translated in code

| Artifact | Language |
|----------|----------|
| Variable / function / file names | English |
| Log keys | English |
| API JSON field names | English |
| Error codes from API (`CITY_MISMATCH`) | English — map to i18n message in UI layer |

## Privacy-sensitive copy

Sensitive category screens must use neutral, non-identifying strings. Author labels follow [privacy-and-identity.md](../system/privacy-and-identity.md) — never expose contributor GPS or real names on sensitive reports.
