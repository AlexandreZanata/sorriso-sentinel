# Technology Stack

Official development stack for Sorriso Sentinel. All new code must align with these choices unless superseded by an ADR.

## Summary

| Layer | Choice | Rationale |
|-------|--------|-----------|
| Language | **TypeScript** | End-to-end types, shared domain models |
| API | **NestJS** | Modules enforce Clean Architecture/DDD; CQRS, queues, RBAC ecosystem |
| Database | **PostgreSQL 18.1** | Native **UUID v7**, RLS, constraints, mature ops |
| Geospatial | **PostGIS** | Spatial queries, GiST indexes, neighborhood polygons |
| Time series | **TimescaleDB** | City timeline, trends, predictions on occurrence events |
| Cache / light queue | **Redis + BullMQ** | Map cache, rate limiting, reputation counters, async jobs |
| Message broker (future) | **RabbitMQ** or **Kafka** | Outbox, cross-service events — not day one |
| Media storage | **S3-compatible** (R2 / MinIO) | Photos with server-side anonymization pipeline |
| Mobile | **React Native + Expo** | iOS/Android, shared types |
| Web / admin | **Next.js** | Public map SSR, SEO, admin dashboard |
| Maps | **MapLibre GL** | Open source, no vendor lock-in |
| Monorepo | **Turborepo** | Shared packages, unified CI |
| Heavy geo workers (optional) | **Go** | Aggregation, pattern detection — separate service via events |

## Backend — API

**NestJS (TypeScript)** is the primary API and application layer:

- Vertical Slice modules per feature
- CQRS for read-heavy map endpoints
- Outbox pattern for domain events → BullMQ workers
- Stateless — horizontal scaling behind load balancer

**Go** is reserved for CPU-heavy geoprocessing if needed later (neighborhood health aggregation, recurrence detection at scale). Communicates via events — not embedded in the NestJS process.

## Database — PostgreSQL 18.1

### UUID v7 (native)

All primary keys use **UUID v7** generated by PostgreSQL 18.1 natively:

```sql
-- Example (exact syntax per PG 18 uuidv7() function)
id UUID PRIMARY KEY DEFAULT uuidv7()
```

Benefits: time-ordered IDs, better B-tree locality than UUID v4, no application-side ID generation required.

### Extensions

| Extension | Purpose |
|-----------|---------|
| `postgis` | Geometry columns, spatial indexes, distance/within queries |
| `timescaledb` | Hypertables for occurrence event time series |
| `pgcrypto` | Sensitive field encryption |

### Row Level Security

RLS is mandatory for:

- City/tenant isolation
- Privacy levels (public, neighborhood, approximate, hidden)
- Sensitive category access restrictions

### Graph modeling

Territorial relationships (School → Street → Occurrences) use **relational tables + recursive CTEs**. No Neo4j until graph queries become the primary bottleneck.

## Cache and queues

| Tool | Use |
|------|-----|
| **Redis** | Map tile cache, rate limiting, real-time reputation counters |
| **BullMQ** | Outbox dispatch, media anonymization, trend jobs, mission generation |

Start with BullMQ on Redis. Introduce RabbitMQ/Kafka only when multi-region or multi-service event routing is required.

## Media pipeline

```text
Client upload → API (pre-signed URL) → Worker:
  1. Strip EXIF/GPS/device metadata
  2. Validate content
  3. Persist to S3-compatible storage
  4. Emit MediaAnonymized event
```

Never trust client-side stripping alone.

## Frontend

| App | Stack | Package |
|-----|-------|---------|
| Mobile | React Native + Expo | `apps/mobile` |
| Web / map / admin | Next.js | `apps/web` |
| Shared types + validation | TypeScript | `packages/domain`, `packages/shared` |

Map rendering: **MapLibre GL** (open source fork of Mapbox GL).

## Authentication

| Method | Default |
|--------|---------|
| Local cryptographic identity (device key pair) | Yes — anonymous/pseudonym |
| Email/password (optional) | Public mode only |
| OAuth providers | Optional — never the only path |

JWT: short-lived access tokens + separate revocable refresh tokens.

## Observability

- Structured JSON logs with correlation ID
- Health endpoints: `/health`, `/ready`, `/live`
- Metrics: latency, error rate, throughput per route
- Distributed tracing when workers split to Go

## Local development (planned)

```text
docker compose:
  - postgres:18 (PostGIS + TimescaleDB)
  - redis:7
  - minio (S3-compatible)
  - api (NestJS)
  - worker (BullMQ consumers)
```
